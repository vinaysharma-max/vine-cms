## Base URL

All public content requests go to:

```text
https://vinecms.tech/api/public/v1
```

## API Versioning

The current stable public API version is `v1`. Use the explicit versioned route format:

```text
https://vinecms.tech/api/public/v1/{API_KEY}/{resource}
```

Example:

```text
https://vinecms.tech/api/public/v1/{API_KEY}/posts
```

## Authentication

The Vine public API uses path-based API key authentication.

### Getting Your API Key

Create or copy an API key from your workspace inside Vine.

### Using Your API Key

Replace `{API_KEY}` with the real key value:

```bash
# Before
https://vinecms.tech/api/public/v1/{API_KEY}/posts

# After
https://vinecms.tech/api/public/v1/your-real-api-key/posts
```

## Making Requests

No special headers are required.

You can call the API from:

- **Browser**: Use `fetch()` or `XMLHttpRequest`
- **cURL**: Use the command line examples provided in each endpoint
- **Any HTTP client**: Standard GET requests work out of the box

### Example Request

Here's a simple example using cURL:

```bash
curl "https://vinecms.tech/api/public/v1/{API_KEY}/posts?page=1&pageSize=20"
```
or using JavaScript:

```js
const response = await fetch(
  'https://vinecms.tech/api/public/v1/{API_KEY}/posts?page=1&pageSize=20'
);

if (!response.ok) {
  const error = await response.json();
  throw new Error(error.error || 'Request failed');
}

const data = await response.json();
console.log(data.posts);
console.log(data.pagination);
```

### Example Response

```json
{
  "ok": true,
  "posts": [],
  "pagination": {
    "page": 1,
    "pageSize": 20,
    "totalItems": 0,
    "totalPages": 0,
    "hasNextPage": false,
    "hasPreviousPage": false
  }
}
```

## Pagination

Collection endpoints support optional query parameters:

- `page`: positive integer page number
- `pageSize`: positive integer page size, up to `100`

Example:

```bash
curl "https://vinecms.tech/api/public/v1/{API_KEY}/posts?page=2&pageSize=10"
```

If you omit both parameters, Vine preserves the current `v1` behavior and returns the full collection while still including pagination metadata in the response.

## Response Format

All successful responses return JSON data. Error responses also use JSON with an `error` field describing the issue.


### Success Response

```json
{
  "ok": true,
}
```

### Error Response

```json
{
  "ok": false,
  "error": "Invalid API key"
}
```

## Health And Status Checks

Use these endpoints to verify the API surface is live:

```bash
curl "https://vinecms.tech/api/health"
curl "https://vinecms.tech/api/public/v1/status"
```
